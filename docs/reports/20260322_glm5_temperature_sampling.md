## 深度分析：为什么会这样？

### 1. Temperature 的作用机制

Temperature 通过缩放 logits 来控制分布的"尖锐程度"：

```
P(token_i) = softmax(logits / T)
```

- **T→0**（贪心）：logits 差距被无限放大，永远选 argmax，输出完全确定
- **T=0.8**（本实验）：logits 差距被适度放大（除以 0.8 = 乘以 1.25），分布比 T=1.0 **更尖锐**，top-1 概率略微升高
- **T=1.0**：直接用原始 logits 对应的概率，模型"原始意图"
- **T>1**：logits 差距被压缩，分布趋于平坦，低概率 token 被选中的机会增加

**采样方式**：不论 T 取何值，最终都是按概率分布做**加权随机抽样**，相当于一个加权转盘——概率越高的 token 占据转盘越大的扇区，但每次拨动转盘都有随机性：

```
┌──────────────────────────────────────────┐
│█████████████████████████████ 主动  74.5% │  ← 本实验请求 2 命中
│███ 简要  7.8%                            │  ← 本实验请求 1 命中（低概率被选中）
│███ 简单  6.9%                            │
│█ 适  2.9%                                │
│ 其余 7.9%                                │
└──────────────────────────────────────────┘
```

本实验中 top-1（`主动`）概率为 74.5%，意味着**每次采样仍有约 25.5% 的概率不选它**。请求 1 恰好命中了 Rank-2（`简要`，仅 7.8% 概率）。

### 2. 为什么两次请求 logits 完全相同？

两次请求的 prompt、模型、tokenization 完全一致，模型的 forward pass 是确定性的（deterministic）。在相同的 KV cache 和 context 下，target model 输出的 logits 必然相同。

**不同的只是随机数种子**——用来从概率分布中抽样的随机数。

### 3. 分歧的"蝴蝶效应"

分歧发生在第 44 个 token，而这个位置属于 reasoning 阶段的内部思考。

- 请求 1 选择了 `简要` → 后续生成了"你好！我是GLM，Z.ai训练的大语言模型。很高兴能与你交流！

我可以回答问题、提供信息或协助你完..."
- 请求 2 选择了 `主动` → 后续生成了"你好！很高兴见到你。我是GLM，Z.ai训练的大语言模型。

今天我能为你做些什么？无论是回答问题、..."

一个 token 的差异，导致后续整个回答的结构和表达方式都发生了变化，最终输出长度也不同（100 vs 72 字符）。

### 4. EAGLE Speculative Decoding 下的 Accept Rate 对比

| 指标 | 请求 1 | 请求 2 |
|------|--------|--------|
| Cycles | 59 | 55 |

## 结论

本实验从 logits 层面直接证明了 **temperature 采样的随机性本质**，并揭示了 **thinking model 的双层随机性**：

1. **相同 logits，不同结果**：模型的 logits 计算是确定性的，随机性完全来自采样过程中的随机数种子
2. **分歧发生在 reasoning 阶段**：第 44 个 token（`简要` vs `主动`）位于模型的内部思考过程中，而非最终输出。这说明 thinking model 的随机性有两层：
   - **Reasoning 层**：思考策略的随机选择（"简要介绍自己" vs "主动询问需求"）
   - **Output 层**：基于不同思考策略生成的不同输出内容
3. **分歧点"一击必发"**：在概率为 74.5%（top-1）的位置，依然有 25.5% 的概率产生分歧
4. **分歧的级联效应**：reasoning 阶段的单个 token 随机选择，通过 KV cache 的积累，导致后续所有 token（包括 reasoning 和 output）的生成路径彻底不同
5. **对 EAGLE 的影响**：output 内容的差异直接影响了 draft model 的预测准确率（accept rate 49.7% vs 48.5%）

**工程启示**：
- 在需要可复现性的场景（如 A/B test、debug）中，必须固定随机数种子
- 在需要多样性的场景（如创意生成），temperature > 0 的采样是简单有效的手段
- 对于 thinking model，reasoning 阶段的随机性会被放大到 output 阶段，即使 reasoning 差异很小（一个词），最终输出也可能完全不同
