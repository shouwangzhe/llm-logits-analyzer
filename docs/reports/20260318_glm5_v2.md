## 重建逻辑

### 数据结构

```
target[0]:   pos=0, verified token（上一 cycle 的实际输出），accept=None
target[1]:   pos=1, 验证 draft[1]，accept=True/False
target[2]:   pos=2, 验证 draft[2]，accept=True/False
target[3]:   pos=3, bonus 位置，is_bonus=True
draft[0..2]: 3 个 draft 预测 token
```

### 重建公式

```
每个 cycle 的输出 tokens =
    draft[0..accept_length-1]      (已接受的 draft tokens)
  + target[accept_length].topk[0]  (bonus = 该位置的 argmax 预测)
```

- `target[0]` 是上一 cycle 的实际 bonus（verified token），不参与 accept/reject
- 接受规则：`draft[i]` 与 `target[i+1].topk[0]` 比较
- Bonus token 取 `target[accept_length].topk[0]`（greedy argmax）

### 关键发现：Speculative Sampling

**Cycle 46** 揭示了服务器使用 speculative sampling（非 greedy）：

```
Cycle 46 (accept=0):
  draft[0]='   ' → target[1].top1=' *'  p=0.999938  draft≠top1 → REJECT
  draft[1]=' *'  → target[2].top1='  '  p=0.994369  (不到达此步)
  draft[2]='  '  → target[3].top1=' **' p=0.732311  (不到达此步)
  bonus = target[0].topk[0] = '   ' (argmax, p=0.90)
```

但 target[0] 在数据中显示 token_text='   '，top1_prob=0.904，
下一个 cycle 47 的 verified (target[0]) = '  '（不是 '   '）。

这说明：**实际 bonus token 是通过 sampling 决定的，不一定等于 argmax**。
用 argmax 重建会在 bonus 位置引入误差，这解释了 ~17.8% 的重建差异。

## Cycle 44 — Logits 深度分析

以 Cycle 44 为例，展开到 logits 层面，分析每个 token 的生成决策。

**基本信息**: seq_len=15, accept_length=3, draft=[`'.'`, `' **'`, `'分析'`], bonus=`'用户'`

### EAGLE Accept 规则（Speculative Sampling）

```
accept_prob = min(1, p_target(draft_token) / p_draft(draft_token))
若 random() < accept_prob → ACCEPT，否则 REJECT
```

Greedy 时：`accept_condition = (draft_token_id == argmax(target_logits))`

## 结论

### 1. EAGLE 推理机制验证

- **Draft-Verify 流程正确**：Draft model 预测 N 个 tokens，Target model 逐个验证
- **Speculative Sampling**：服务器使用 temperature>0 的 speculative sampling（非 greedy）
- **接受率**：57.3% 的 draft tokens 被接受，属于 EAGLE 有效推理的正常水平
- **Bonus token**：每个 cycle 在 accept_length 位置取 target argmax 作为重建 bonus

### 2. 数据完整性

- **Cycle 数据覆盖**：110 个 cycles，生成 299 个 tokens
- **预填充 token 缺失**：首 token `'1'` 在预填充阶段生成，不在 cycle 数据中
- **Draft logits**：`--disable-cuda-graph` 下完整保存，shape=(2, vocab_size)（步骤 0 和 1）
- **重建准确率**：82.2% 字符完全匹配（difflib ratio=0.812）

### 3. 差异原因

| 原因 | 字符数 | 说明 |
|---|---|---|
| 预填充 token 缺失 | ~1 | 首 token `'1'` 不在 cycle 数据中 |
| Speculative sampling bonus 偏差 | ~73 | 实际 bonus 由采样决定，与 argmax 不同 |
| Tokenizer 多字节乱码 | ~5 | 多字节汉字 byte-level token 单独解码 |
| 语义路径差异 | ~10 | 同义词替换（`'佩亚诺'` vs `'皮亚诺'`） |

### 4. 关键发现

**CycleCollector 采集逻辑正确**，82.2% 字符匹配证明了：
1. Draft tokens 和 target logits 被正确捕获
2. Accept/reject 逻辑正确反映了实际推理过程
3. Bonus token 机制正确（argmax 重建存在 ~17% speculative sampling 误差）

**更精确的重建需要 temperature=0（greedy）**：在 greedy 模式下：
- `accept = (draft_token_id == argmax(target_logits))`
- bonus token = argmax(target_logits)
- 重建准确率可接近 100%（除预填充 token 缺失外）

**Speculative Sampling 的影响**：bonus token 位置的 logit 分布越平坦（如 cycle 44 pos=3，top3 logit 差距 < 0.5），argmax 重建越不可靠，因为任何 top-k 候选都有可能被采样选中。
