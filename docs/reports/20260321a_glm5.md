## 结论

### 1. EAGLE 机制验证

- **accept=3 占比 40.8%**（204/500），是最常见的接受结果，说明 draft model 在高确定性位置命中率高
- **Output 阶段接受率（67.8%）> Thinking 阶段（57.4%）**：输出阶段内容结构化，draft model 预测更准
- **Draft top1_prob 普遍极低**（多数 < 0.001），但 accept/reject 由 target 验证决定，与 draft 置信度无关

### 2. 重建准确率分析

- **Greedy target 重建匹配率 22.6%**，内容高度相似但位置有偏移
- 偏移根因：boundary cycle 的 bonus token 因 temperature > 0 采样了 rank-2 token（`'###'`，prob=0.311），而非 argmax（`'这是一个'`，prob=0.353）
- **18号数据 82.4% 是偶然**：boundary bonus 恰好 argmax = 实际采样值
- **数据本身正确**：`'###'` 在 cycle 349 bonus topk rank-2，验证了 target logits 完整捕获了实际输出 token

### 3. 工具改进建议

- **CycleCollector 改进**：在 `on_verify_done` 时额外传入实际采样的 token ids（`output_ids`），记录到 `actual_output_token` 字段，替代当前的 argmax 记录
- 改进后可实现精确重建，不受 temperature 影响
