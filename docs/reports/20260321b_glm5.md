### EAGLE Greedy Accept 规则

```
accept_condition: draft_token_id == target_verified_id[pos]
```

即：target model 在该位置采样/选择的 token == draft token → **接受**；否则 → **拒绝**，并将 target 实际选择的 token 作为 bonus token 输出，终止本 cycle。

### 关键观察

1. **Accept 规则基于 verified_id**：`draft_token_id == target_verified_id` 才接受，支持 stochastic sampling（非严格 greedy argmax）
2. **Target 极度确定的位置** (pos=0, prob=0.999999): logit 差距 15.0，draft 只要猜对就必然接受
3. **Target 中等确定的位置** (pos=1,2, prob≈0.85): 存在竞争候选，但 top-1 明确，draft 猜对则接受
4. **Bonus token 不确定性高** (pos=3, prob=0.43): 三个候选 logit 差距 < 0.5，模型在此处有多种合理选择，这类位置是 draft model 最难预测的
5. **Draft 与 Target 对齐程度**: Step 1 top-1 与 target pos 2 top-1 完全一致，说明 draft model 在这个上下文中学到了正确的分布
6. **verified_id fix 的重要性**: 在本 cycle 中 bonus token 的 verified_id 与 top-1 一致（均为 `'用户'`），但在其他 cycle（如 Cycle 338）中，verified_id=`'需求'` 而 top-1=`'   '`，此差异正是旧报告 82.4% 匹配率 vs 本报告 100% 匹配率的根本原因
